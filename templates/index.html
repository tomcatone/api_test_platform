        前置 SQL（HTTP 請求前執行）
        前置SQL 提取變量 — 執行後將查詢結果注入為 {{變量名}}，可在 HTTP 請求 Body/Headers/URL 中引用
        後置 SQL（HTTP 請求後執行）
        後置SQL 提取變量 — 執行後將查詢結果注入為 {{變量名}}，可在後續接口中引用
        前置 Redis 取值 — 請求前從 Redis 讀取 key，注入為 {{變量名}} 供 Body/Headers/Params 使用
        典型場景：驗證碼存在 Redis，讀出後填入請求體
        Key 支持 {{變量名}}，如 sms:{{mobile}}；value 是 JSON 時可指定提取字段
        SSL / HTTPS 驗證設置
        勾選=跟隨 301/302 重定向（默認）；取消勾選=停在原始響應，可獲取 Location 頭
        OAuth2 Authorization Code Flow 設定 requests-oauthlib Spring Security 兼容
        自動推導端點：{base_url}/oauth2/authorize　·　 {base_url}/login　·　{base_url}/oauth2/token
        oauth.verify = True 正常驗證服務端 SSL 憑證（生產環境推薦）
        oauth.verify = False 跳過 SSL 驗證 + 自動靜音 InsecureRequestWarning（本地/自簽名環境）
        執行流程： ① POST {base_url}/login（模擬登錄，獲取 Session Cookie）→ ② GET {base_url}/oauth2/authorize（攜帶 Cookie 自動授權，獲取 code）→ ③ POST {base_url}/oauth2/token（換取 access_token）→ ④ 用帶 Bearer Token 的 Session 發送目標 API 請求
        requests verify 參數 直接傳給 requests
        等價於 requests.get(url, verify=True)　·　 優先於下方「SSL 驗證模式」選單生效
        跳過 SSL 驗證存在安全風險，僅建議在內網測試環境使用。
